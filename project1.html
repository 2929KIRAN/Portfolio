Transaction Data Analysis

Examining transaction data- looking for inconsistencies, missing data across the data set, outliers, correctly identified category items, numeric data across all tables. Determining any anomalies and make the necessary changes in the dataset and saving it. Having clean data will help when it comes to analysis.

Examining customer data – checking for similar issues in the customer data, looking for nulls and merging the transaction and customer data together so it’s ready for the analysis ensuring save the files along the way.

Data analysis and customer segments – in analysis making sure to define the metrics – looking at total sales, drivers of sales, where the highest sales are coming from etc. Exploring the data, creating charts and graphs as well as noting any interesting trends and/or insights found.

Deep dive into customer segments – defining recommendation from the insights, determining which segments should be targeting, if packet sizes are relative and form an overall conclusion based on analysis.

Project Goals

  • Analyze transaction and customer data to identify trends and inconsistencies.
  • Develop metrics and examine sales drivers to gain insights into overall sales performance.
  • Create visualizations and prepare findings to formulate a clear recommendation for the client's strategy.
